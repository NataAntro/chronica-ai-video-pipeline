# Архитектура

## Границы модулей

- `providers/` реализует внешние границы. Локальные реализации работают без сети; HTTP-реализации включаются явно.
- `domain/` содержит типы, runtime-схему недоверенного LLM-ответа и JSON Schema для Structured Output.
- `prompts/` хранит версионируемые generate/repair-контракты.
- `evaluation/` отделяет оценку редакционного качества от структурной валидации.
- `services/` выполняет однотипные этапы: recovery, chunking, TTS, WAV aggregation, subtitles, render и manifest.
- `pipeline/` связывает этапы и отвечает за каталог конкретного запуска.
- `remotion/` получает только сериализуемые `RenderProps` и не знает о providers.

## Обработка ошибок

Невалидный или не прошедший quality gate сценарий не доходит до TTS. Orchestrator делает не более трёх попыток: сначала `generate`, затем `repair` с ошибками схемы или оценки. Каждая попытка попадает в трассу. Сетевая ошибка provider также попадает в bounded recovery. После исчерпания попыток pipeline завершается с явной ошибкой.

После каждого завершённого этапа конвейер атомарно обновляет манифест. Повтор с тем же `RUN_ID` проверяет размер и SHA-256 артефактов, а также fingerprint семантических входов этапа. Валидные результаты переиспользуются; повреждение файла или смена темы, prompt/model/TTS profile выполняют только затронутый этап и его зависимости. Поле `attempts` увеличивается только при фактическом запуске этапа, поэтому восстановление можно отличить от повторного вычисления.

Граф зависимостей описан отдельно от последовательности выполнения. Решение для этапа фиксируется как `reused` или `executed` вместе с причиной: проверенные артефакты, отсутствующий файл, нарушение целостности, незавершённый статус или изменение вышестоящего этапа. Это позволяет диагностировать восстановление без анализа журналов процесса.

## Timing

Каждый фрагмент TTS проходит разбор RIFF/WAVE chunks и проверку PCM-параметров. `durationMs` вычисляется из размера `data` payload, sample rate, channels и bit depth. Разметка размещает субтитры последовательно и без перекрытий; последний `endMs` определяет длительность Remotion-композиции. Внешний TTS имеет явный тайм-аут, а его HTTP metadata не используется как источник истины о длительности.

## Доверительные границы

- LLM JSON — недоверенный ввод, проверяется Zod.
- TTS WAV — недоверенный бинарный ввод, проверяются RIFF/WAVE, границы chunks, PCM format и фактическая длительность.
- Секреты существуют только в окружении процесса и не записываются в manifest.
- `artifact-manifest.json` содержит только относительные пути, размеры и SHA-256.
- `RUN_ID` валидируется до создания каталога и не допускает path traversal.
